A `radio` component enables users to choose one option from a predefined set. It ensures only one selection can be active at a time within a group.

## Component Usage
### Default

A default `radio` component is pre-selected to indicate a standard choice in a group of options. This helps guide users by providing a recommended selection right from the start.

<SRadio client:only="vue" />
<Code code={SRadioRaw} lang="vue" title="default.vue" />

### Disabled

A disabled `radio` component is inactive and cannot be selected, indicating unavailable options within a group.

<SRadioDisabled client:only="vue" />
<Code code={SRadioDisabledRaw} lang="vue" title="disabled.vue" />

### Invalid

An invalid `radio` component indicates that the selected option does not meet the required criteria. This prompts users to correct their choice before proceeding.

<SRadioInvalid client:only="vue" />
<Code code={SRadioInvalidRaw} lang="vue" title="invalid.vue" />

### Required

A required `radio` component mandates that users must select an option before submitting the form. This ensures that a choice is made, preventing incomplete submissions.

<SRadioRequired client:only="vue" />
<Code code={SRadioRequiredRaw} lang="vue" title="required.vue" />

### Custom Validator

A custom validator radio component lets developers set specific validation rules for user selections, enabling tailored error messages based on application needs.
